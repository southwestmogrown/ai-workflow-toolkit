commit message cheat sheet

structure

type(scope)!: short description
↑ required ↑ optional ↑ breaking ↑ required, imperative, ≤72 chars

types

feat
new feature or capability
fix
bug fix
chore
tooling, config, deps — no production code change
docs
documentation only
refactor
restructure without behavior change
test
add or update tests
perf
performance improvement

examples

✓ good feat(coach): add Supabase pgvector similarity search
✓ good fix(auth): redirect loop on expired session token
✓ good chore: upgrade next to 15.2.1
✓ good — breaking change feat(api)!: rename /lessons endpoint to /modules
✗ too vague fix stuff
✗ past tense added socratic coach feature
✗ too long feat: add the new socratic coaching mode to quizquest with supabase vector search integration and claude api

rules

→Subject line: imperative mood ("add", not "added" or "adds")
→Subject line: ≤72 characters, no period at end
→Scope: the module, page, or domain affected — optional but useful
→Body (optional): blank line after subject, then explain the WHY not the WHAT
→Breaking changes: append ! after type/scope, explain in body
→Reference issues: close with Closes #42 or Refs #42 in body
→One logical change per commit — don't bundle unrelated changes
