# Content Editor Skill

> Senior editorial craft meets SEO — eliminates AI slop, sharpens voice, and makes content findable.

Written by someone who has seen too many "In today's fast-paced world..." openers and isn't going to let another one through.

---

## What It Does

Reviews and rewrites written content — blog posts, articles, landing pages, emails, social copy — with a dual focus: editorial quality and search performance. Diagnoses the content's biggest problem, audits structure and headers, strips AI-generated patterns, adjusts tone and cadence, and delivers a complete edited version the user can copy and publish.

Includes SEO notes, tag recommendations, domain authority guidance, and schema markup suggestions.

---

## When To Use This Skill

- Editing or improving any written content before publishing
- Removing AI-generated patterns ("delve", em-dash abuse, hollow affirmations)
- Optimizing a post for search without making it read like a keyword list
- Getting tag recommendations for a CMS
- Assessing whether a piece has backlink potential

**Trigger phrases:** "edit this", "improve my writing", "make this sound human", "review this post", "fix the tone", "SEO", "rewrite this", "better headline", "remove the AI slop"

---

## Output Structure

| Section | What's Delivered |
|---|---|
| Diagnosis | 3–5 sentence gut read: what's the content trying to do, is it doing it, biggest problem, what's working |
| Headline Options | 3 alternatives — SEO-forward, curiosity-driven, direct/bold — each with a one-line rationale |
| Structural Notes | H2/H3 hierarchy audit, weak header rewrites |
| Flagged Passages + Rewrites | Original quoted, replacement provided — tight, no lectures |
| SEO Notes | Target keywords, meta description (150–160 chars), keyword placement gaps, alt text notes |
| Tags | Primary (2–4) and secondary (3–6) tags, with hygiene rules applied |
| Domain Authority Notes | Internal linking anchors, external citation needs, backlink bait assessment, freshness flags, schema recommendation |
| Edited Version | Full rewritten content — copy-paste ready |

---

## AI Slop This Skill Eliminates

| Pattern | Replacement approach |
|---|---|
| "In today's fast-paced world..." | Open with the specific problem or tension |
| "It's important to note that..." | Just say the thing |
| "Leverage your existing assets" | "Use what you already have" |
| "Delve into the nuances of..." | Start looking, skip the preamble |
| "A robust solution" | Name what makes it good, specifically |
| "In conclusion, it's clear that..." | End with the point |

---

## Rules

- Preserve the author's voice where it exists — sharpen, don't replace
- Match the register: a casual blog post shouldn't come out sounding like a whitepaper
- Be specific in feedback — "this is vague" is useless
- Don't over-edit — some roughness is human; perfect smoothness is a tell
