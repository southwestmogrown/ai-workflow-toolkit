# Claude / AI Artifact Remover

Systematically find and remove AI development artifacts — files, comments, code
patterns, and config — that are safe (or necessary) to strip before production.

**Always run a dry-run scan first. Never delete without showing the user the hit list.**

---

## Step 1: Scan — find everything

Run the full scan from the project root. Adjust paths if the user specifies a
subdirectory.

```bash
# ── 1a. AI config & instruction files ───────────────────────────────────────
find . \
  -not -path '*/node_modules/*' \
  -not -path '*/.git/*' \
  -not -path '*/vendor/*' \
  \( \
    -name "CLAUDE.md" \
    -o -name ".cursorrules" \
    -o -name ".cursorignore" \
    -o -name ".github/copilot-instructions.md" \
    -o -name "copilot-instructions.md" \
    -o -name ".aider*" \
    -o -name "aider.conf*" \
    -o -name ".continue" \
    -o -name "*.prompt.md" \
    -o -name "*.prompts.md" \
    -o -name "AI_NOTES.md" \
    -o -name "AI_CONTEXT.md" \
    -o -name "DEV_NOTES.md" \
    -o -name "COPILOT_NOTES.md" \
  \) \
  -print

# ── 1b. Hardcoded API keys / secrets (scan only — never auto-delete) ─────────
grep -rn \
  --include="*.ts" --include="*.tsx" --include="*.js" --include="*.jsx" \
  --include="*.py" --include="*.env*" --include="*.json" \
  --exclude-dir=node_modules --exclude-dir=.git \
  -E "(sk-[a-zA-Z0-9]{20,}|ANTHROPIC_API_KEY\s*=\s*['\"][^'\"]+['\"]|OPENAI_API_KEY\s*=\s*['\"][^'\"]+['\"]|claude-[a-z]+-[0-9]{8})" \
  . 2>/dev/null

# ── 1c. AI-generated comment patterns ────────────────────────────────────────
grep -rn \
  --include="*.ts" --include="*.tsx" --include="*.js" --include="*.jsx" \
  --include="*.py" \
  --exclude-dir=node_modules --exclude-dir=.git \
  -iE "(generated by (claude|copilot|chatgpt|cursor|ai)|TODO: (claude|ai|copilot)|NOTE: (claude|ai)|this (code|function|component) was (generated|written|created) by|claude: |copilot: )" \
  . 2>/dev/null

# ── 1d. Debug / test prompts hardcoded in source ─────────────────────────────
grep -rn \
  --include="*.ts" --include="*.tsx" --include="*.js" --include="*.jsx" \
  --include="*.py" \
  --exclude-dir=node_modules --exclude-dir=.git \
  -iE "(system:\s*[\"']you are a helpful|const (testPrompt|debugPrompt|devPrompt|aiPrompt)\s*=|\"role\":\s*\"system\".*helpful assistant)" \
  . 2>/dev/null

# ── 1e. Claude model strings (hardcoded, should use env vars) ────────────────
grep -rn \
  --include="*.ts" --include="*.tsx" --include="*.js" --include="*.jsx" \
  --include="*.py" \
  --exclude-dir=node_modules --exclude-dir=.git \
  -E "\"claude-(opus|sonnet|haiku)-[0-9]" \
  . 2>/dev/null

# ── 1f. Oversized AI-generated comments (blocks > 8 lines starting with //) ──
grep -rn \
  --include="*.ts" --include="*.tsx" --include="*.js" --include="*.jsx" \
  --exclude-dir=node_modules --exclude-dir=.git \
  -c "^\s*//" \
  . 2>/dev/null | grep -v ":0$" | sort -t: -k2 -rn | head -20
```

## Step 2: Categorize hits

Group every finding into one of three buckets before touching anything:

| Bucket | Action | Examples |
|--------|--------|---------|
| **Safe to delete** | Remove after user confirms | `CLAUDE.md`, `.cursorrules`, AI-generated comment blocks, `*.prompt.md` |
| **Needs edit** | Show user, edit in place | Hardcoded model strings → env var, debug system prompts → parameterized, oversized comment blocks → trim |
| **Flag only** | Report, let user decide | Suspected hardcoded API keys (never auto-delete), large generated comment blocks in critical files |

**Never silently delete anything.** Always show the full hit list, grouped by bucket, and wait for confirmation before proceeding.

---

## Step 3: Present the hit list

Format the report like this before taking any action:

```
── AI ARTIFACT SCAN RESULTS ─────────────────────────────────────

SAFE TO DELETE (confirm to remove)
  ./CLAUDE.md                          AI instruction file
  ./.cursorrules                       Cursor IDE config
  ./src/utils/test.prompt.md           Dev prompt file

NEEDS EDIT (will show diffs)
  ./src/lib/ai.ts : 12                 Hardcoded model string → use process.env.CLAUDE_MODEL
  ./src/components/Chat.tsx : 44       Debug system prompt → should come from config
  ./src/hooks/useCoach.ts : 8-31       24-line AI-generated comment block → trim to 3 lines

FLAG ONLY (review manually)
  ./.env.local : 3                     Possible API key — verify it's in .gitignore
  ./src/api/route.ts : 67              "Generated by Claude" comment — harmless but tells the world

────────────────────────────────────────────────────────────────
```

Ask: "Shall I proceed with the safe-to-delete items, then show diffs for the edits?"

---

## Step 4: Execute — safe deletes

Only after explicit user confirmation:

```bash
# Remove confirmed files (example — build this list from actual scan results)
rm ./CLAUDE.md
rm ./.cursorrules
rm ./src/utils/test.prompt.md

# Verify nothing critical was lost
git diff --stat HEAD
```

---

## Step 5: Execute — edits

For each "needs edit" item, show the before/after diff and confirm before writing:

### Hardcoded model string → env var

```typescript
// BEFORE
const response = await anthropic.messages.create({
  model: "claude-sonnet-4-5-20251022",   // ← hardcoded, will break on model update
  ...
});

// AFTER
const response = await anthropic.messages.create({
  model: process.env.CLAUDE_MODEL ?? "claude-sonnet-4-5-20251022",
  ...
});
```

Add to `.env.example`:
```
CLAUDE_MODEL=claude-sonnet-4-5-20251022
```

### Debug system prompt → config-driven

```typescript
// BEFORE
const systemPrompt = "You are a helpful assistant. This is a test.";

// AFTER
const systemPrompt = process.env.SYSTEM_PROMPT
  ?? config.ai.systemPrompt
  ?? defaultSystemPrompt;
```

### Oversized AI comment block → trim

Keep the first 2–3 lines that describe what the function does. Cut the rest.
AI-generated comments often explain *how* obvious code works — that's noise.
Good comments explain *why*, not *what*.

## Step 6: Verify .gitignore coverage

Ensure these patterns exist in `.gitignore` so they never accidentally ship again:

```gitignore
# AI development artifacts
CLAUDE.md
.cursorrules
.cursorignore
.aider*
aider.conf*
*.prompt.md
*.prompts.md
AI_NOTES.md
AI_CONTEXT.md
COPILOT_NOTES.md
.github/copilot-instructions.md

# Local env (should already be here)
.env.local
.env*.local
```

Show the user what's missing from their current `.gitignore` and offer to add it.

---

## Step 7: Final confirmation

Run a second pass of the scan to verify the project is clean:

```bash
# Should return zero results after cleanup
find . \
  -not -path '*/node_modules/*' \
  -not -path '*/.git/*' \
  -name "CLAUDE.md" -o -name ".cursorrules" -o -name "*.prompt.md" \
  | wc -l
```

Report: "X artifact(s) removed, Y edits made, Z items flagged for manual review."

---

## What this skill does NOT remove

These are legitimate in production and should be left alone:

- `CLAUDE.md` files inside `.github/` that are genuinely part of repo documentation
- AI feature code itself (chat UI, API calls, prompt logic) — that's the product
- Type definitions or imports from `@anthropic-ai/sdk` — those are runtime deps
- Comments that explain business logic, even if AI-assisted writing
- Test files that test AI features

When in doubt, flag rather than delete.
