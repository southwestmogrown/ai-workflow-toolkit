# CLAUDE.md Generator Skill

> Generate a production-ready `CLAUDE.md` that gives Claude accurate, lasting context for any project — in one session.

A `CLAUDE.md` is the single highest-leverage thing you can add to a repository that uses Claude as a coding collaborator. This skill writes one correctly, the first time.

---

## What It Does

Gathers project context — stack, architecture, directory structure, naming conventions, anti-patterns, and agent instructions — then produces a complete, filled-in `CLAUDE.md` with no placeholder text. Claude reads this file automatically at the start of every session, eliminating the need to re-establish context and producing more consistent, convention-aware output.

---

## When To Use This Skill

- Setting up Claude for a new or existing project
- Current Claude sessions keep making wrong assumptions about your codebase
- Onboarding a coding agent to a repo it's never seen
- Preparing a project for public or team-wide AI-assisted development

**Trigger phrases:** "create a CLAUDE.md", "set up AI context for my project", "Claude keeps getting my stack wrong", "configure Claude for agent workflows", "I want better results from Claude in my repo"

---

## What Gets Generated

| Section | Contents |
|---|---|
| Project Overview | Name and one-line description |
| Stack | Full tech table — framework, language, database, styling, testing, package manager, runtime |
| Architecture | 2–4 sentence structural description |
| Directory Structure | Key directories and what lives in each |
| Conventions | Naming, components/modules, state management, error handling, styling |
| Anti-Patterns | Checklist of things Claude must never do without explicit instruction |
| Agent Instructions | Scope, completion criteria, commit style, and guardrails for agent workflows |
| Running the Project | Install, dev, test, and build commands |
| Notes | Quirks, environment variables, deployment targets |

---

## Process Overview

| Step | Action |
|---|---|
| 1 | Gather context — extract from conversation or files, ask only for what's missing |
| 2 | Write the CLAUDE.md — fill every applicable section, remove inapplicable ones, zero placeholder text |
| 3 | Deliver — present in a code block, save as a file if `present_files` is available |

---

## Rules

- Never leave placeholder text in the output — fill it in or remove the section entirely
- Keep the Anti-Patterns section as a checklist — it's easier to scan
- Prefer concrete specifics over vague generalities: "Use named exports" beats "follow good export practices"
- The output should read like it was written by the lead developer, not generated by AI
