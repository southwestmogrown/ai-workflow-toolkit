## Architecture

Pure Markdown repository. No dependencies, no build step, no runtime. The repo contains two top-level asset types: **skills** (structured instruction sets for Claude) and **templates** (starter files for users to copy into their own projects). Each skill lives in its own subdirectory under `skills/`.

## Conventions

### Naming
- Skill files: `SKILL_[skill-name].md` — one per directory, lowercase kebab-case suffix.
- Skill directories: `skills/[skill-name]/` — kebab-case, matches the skill file suffix.
- Template files: descriptive filename matching the output artifact (e.g. `CLAUDE.md`).

### Skills
- One skill per directory under `skills/`.
- Every skill file must include: a trigger description, a step-by-step process, an output template, and hard rules.
- Never leave placeholder text in skill output templates — fill it in or remove the section.
- Skills should read as written by an expert practitioner, not generated by AI.

### Templates
- Templates live under `templates/` and contain placeholders for users to fill in.
- Placeholder text uses bracket notation: `[e.g. Next.js 16]`.

## Anti-Patterns

These must never happen without explicit instruction:

- [ ] Do not add code, build tooling, or dependencies to this repository
- [ ] Do not rename skill files away from the `SKILL_[skill-name].md` format
- [ ] Do not add a skill file directly in `skills/` — it must live in its own subdirectory
- [ ] Do not leave placeholder text in delivered skill output
- [ ] Do not refactor the README or other files outside the scope of the current task

---

## Agent Instructions

These rules apply when Claude is assigned to a GitHub issue or automated task:

- **Scope:** Work only within the files relevant to the current issue. Do not audit unrelated files.
- **Completion criteria:** A task is complete when the Markdown file(s) are well-formed, all placeholders are resolved, and naming conventions are followed.
- **Flag before proceeding if:** The task requires adding a new skill or template that wasn't explicitly requested, or if a skill name conflicts with an existing one.
- **Commit style:** Conventional commits — `feat:`, `fix:`, `chore:`, `docs:`.
- **Do not:** Modify `LICENSE`, change `README.md` structure without instruction, or add non-Markdown files.
